# 📜 コード生成ルール集

このドキュメントは、コーディングエージェントに対して**コード生成時に従うべきルール**を明確に伝えるためのものです。保守性・可読性・拡張性を重視したスタイルを定めます。

---

## ✅ 共通ルール（すべての言語に共通）

- **コメントは適切に記述すること**
  - 何の処理か、なぜそうしているかを明確に。
  - 関数やクラスの冒頭には必ずドキュメンテーションコメントをつける。
  - コメントは日本語で記述すること。

- **定数・変数は意味のある名前にすること**
  - 例：`tmp` → `temp_file_path` / `x` → `user_age` など

- **1関数1責務**（1つの関数は1つの目的に絞る）

- **ファイルは役割ごとに分割すること**
  - 例：API処理は `api.ts` に、UIロジックは `ui.tsx` に、型定義は `types.ts` に

- **エラーハンドリングは必ず行うこと**
  - 例：try-catch構文を省略しない、APIレスポンスのバリデーションを入れる
