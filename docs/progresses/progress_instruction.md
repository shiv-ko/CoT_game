# 進捗マークダウン作成 指示書:

## 1. 指示

指定した日の作業進捗をマークダウンファイル形式でまとめる。  
必ず以下の要素を含めること。

- **日付**（`YYYY-MM-DD`）
- **何を行ったか**（タスクや作業内容）
- **何ができたか**（達成事項・成果）
- 必要に応じて「課題」や「次のステップ」も追加可能

## 2. 出力形式

Gemini CLI は以下の構造でマークダウンを出力すること。

# 今日の進捗（YYYY-MM-DD）

## 何を行ったか

- （作業内容を箇条書き）
- どのファイルを編集したか

## 何ができたか

- （成果や完了事項を箇条書き）

## 課題（任意）

- （現状の課題や問題点）

## 次のステップ（任意）

- （これからの予定や対応策）

```

## 3. 注意点

* 作業内容は簡潔かつ具体的に記載する
* 技術的な用語は省略せず記載する
* 1日分の進捗を1ファイルにまとめる

## 具体例（Webアプリ開発進捗）

# 今日の進捗（2025-08-10）

## 何を行ったか
- Firebase Authentication を用いたログイン機能の実装
- ユーザープロフィールページのレイアウト作成（React + TailwindCSS）
- APIエンドポイントのバリデーション処理を追加

## 何ができたか
- メールアドレス・パスワードでのユーザー登録とログインが可能になった
- プロフィール画像と自己紹介文を表示できるページが完成
- 不正なパラメータ送信時にエラーメッセージを返す機能を実装

## 課題
- ログイン後のリダイレクト処理で一部遷移不具合が発生

## 次のステップ
- リダイレクト処理のデバッグ
- プロフィール編集機能の追加
```
